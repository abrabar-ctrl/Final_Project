## Description of Project:  
Hello.
I conceived this project as an animation that would have to contain some sort of a narrative, the important piece being ***"narrative"*** as that would demand a story beyond just "making things move" or "looking pretty", as there are many ways to animate something using python without it having anything to say (changing shapes, colors, patterns, etc).
I first experimented with importing images and creating a sort of slideshow animation, but quickly realized python would be doing the easy lifting and the images I created would have no connection to any sort of skill with code, so that idea was dropped. As I traveled the web, looking for something that would show what I have learned while realistically showing something skillful, as this specific type of interface is quite difficult for me,
I kept coming back to "turtle", as it is not an extremely advanced library to use, it's one of the more simple ones, in fact. I knew I needed something beyond curationg a still image, so I decided to make a lively "creation" of a city. The moon, stars, cityline, bridge, ocean, and birds needed to ***feel*** alive, so I made sure to keep motion in mind once I began with the motion graphics inspired opening ball animation (my personal favorite, and an homage to _[Maurice Binder's opening sequences of many well-known films of the mid-twentieth century](https://youtu.be/oTo3YtPxG5k?si=iVZ3GuDV_e8veM_b)_), then to the twinkling stars, and finally the blinking sign at the end. All of which required "math", "time", and "random", for the moon color changes.
If I had some more time, I definitely would have continued to improve upon the imagery to as advanced as "turtle" would allow, going beyond a one point perspective to a two-point, or perhaps a three-point (visualizing it now, it shouldn't be too challenging, just very time consuming), however I am proud of the final design, which easily took over 70+ hours to finish from concept, troubleshooting, troubleshooting, revisions, revisions, and final completion. I like to think of this "scene" as the first shot of a developing idea I'm currently working on; a literal "intro" to this new world I've come to use. "Coding a scene" is a completely new muscle I've learned to use.
